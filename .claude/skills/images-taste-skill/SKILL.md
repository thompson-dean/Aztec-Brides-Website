# Image-Taste-Frontend: Image-First Design Methodology

## Core Philosophy

The image is the design source. The code is the translation layer.

Do not begin with freeform coding first. Generate premium visual references, deeply analyze them, then implement faithfully.

## Mandatory Workflow

**Always follow this sequence:**
1. **Generate** — Create design reference image(s)
2. **Analyze** — Deeply inspect the generated image
3. **Implement** — Write frontend code based on analysis

Do not skip image generation when image generation is available. Do not begin with code first.

## Default Configuration

- Design variance: 8/10
- Art direction: 8/10
- Implementation clarity: 9/10
- Spacing generosity: 8/10
- Analysis precision: 10/10
- Image generation eagerness: 10/10

## Codex-Specific Rule

Inside Codex environments: generate ONE large, readable section image per section. Do not compress multiple sections into a single small image board. This preserves:
- Text legibility
- Spacing clarity
- Extraction accuracy for faithful implementation

## Deep Analysis (Before Coding)

Before writing a single line of code, systematically inspect the generated image for:
- Typography: font weight, size, tracking, line-height
- Spacing: padding values, gap patterns, section rhythm
- Buttons: shape, size, color, hover intent
- Colors: exact palette, surface hierarchy
- Components: card structure, badge style, input design
- Layout logic: grid structure, asymmetry, alignment
- Mood: what makes this feel expensive vs generic

Do not proceed with vague aesthetic appraisal. Extract specific, implementable values.

## Anti-Patterns to Avoid

- Cropping unclear sections from previously generated images (destroys proportional relationships)
- Compressing many sections into one unreadable board
- Design drift: simplifying distinctive elements into generic templates during coding
- Overcrowded hero sections with excessive detail
- Repetitive card-spam layouts
- Generic AI visual defaults (floating blobs, purple gradients, centered dark heroes)

## Implementation Discipline

The code must:
- Preserve the original design's visual identity
- Match the spacing logic from the image
- Match the typography mood (weight, tracking, size ratios)
- Replicate component styling exactly — no simplification into defaults
- Honor asymmetry, overlap, and depth from the reference

"Design drift" — where distinctive elements become generic during translation — is a failure mode. The final code must look like the image, not like a watered-down template.

## Pre-Output Check

- [ ] Image was generated before coding began
- [ ] Image was deeply analyzed (typography, spacing, components, colors)
- [ ] Each section has its own reference image (not compressed into one)
- [ ] No cropping from previous images
- [ ] Code faithfully implements the image — no design drift
- [ ] No generic AI defaults (dark centered hero, card spam, purple gradients)
- [ ] Spacing, typography, and component styling match the visual reference
